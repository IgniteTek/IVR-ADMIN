In order to get started, you will want to upload some audio files. This can be done by accessing the Upload Menu in the upper right corner of the screen.

Please note the following:
  • PCI Compliance: If your audio files require a PCI compliant environment,Lucid CX has a specific solution to meet this demand. For PCI compliant implementations, files can be transferred to Lucid CX via Amazon S3 API, using credentials Ignite will provide to our PCI certified S3 environment. Lucid CX will strip PCI sensitive data from both audio and transcription data before the data available through the Lucid portal. The original audio file provided by your system is removed from our S3 environment once file processing is complete.
  • Audio format and quality considerations: Lucid CXâ€™s speech recognition engine supports multiple audio file formats, including wav, mp3 and vox. However, audio files with high compression can reduce speech recognition accuracy. For best performance we recommend not compressing the audio to less than 96 kbps.

LucidVueCX allows you to track and pinpoint keywords inside your inbound calls. Add or remove the keywords you would like to track inside the LucidVueCX dashboard. This can be accomplished by clicking on Edit Keywords from the menu in the top right corner.

After you upload your audio file, the system will convert them into text using our advanced speech recognition engine. The conversion process is variable depending on the number and length of files that you have uploaded. However, you can start to see results immediately. You can refresh the data by clicking the Refresh Query button in the View Contacts panel. We will e-mail you when the process is complete. This usually takes anywhere from one hour to a day.
